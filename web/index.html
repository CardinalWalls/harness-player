Browser boundary guardrail

This browser is render/input only: it renders accepted signed envelopes and can preview a human-authored input envelope. It has no /api/* progress path, no MCP command path, no server-authored substitute messages, and no scripted-screen success path.

Accepted envelope provenance

Every accepted card exposes channel, signer_actor_id, signer_role, causation_ids, correlation_id, payload, and valid status.

Local human_input composer

The browser may carry typed human text, but the preview signer remains the human. Empty text and privileged command text are rejected before a success envelope is created.
