## Stable conclusions

- The product is not just a Hermes plugin and not just a skill format.
- Hermes is the first host runtime, not the product identity.
- The product subject combines:
  - capture/provenance
  - shareable package/share system
  - hosted/local management software
- The synthesis method is separate from the product subject:
  - gather open-source test assets
  - define rules/reference layer
  - isolate heavy host boundaries with mocks
  - reconnect to real systems later
- The external shareable object currently converges on:
  - `skill`
  - `profile`
  - `topology`
- Default external packages should not include raw runtime state by default.
- MoonBit-native originality is most promising in:
  - provenance
  - capture normalization
  - package/export logic
  - topology/profile validation
  - Git-native materialization/publishing paths
